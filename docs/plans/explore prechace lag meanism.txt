The Resolution Pathway:

Synchronous or Async? Synchronous but with a critical delay-spreading mechanism:

When a precached target is needed by the hook and not yet cached, the hook fails open (original effect plays) and queues the name for main-thread resolution.
The Resolution Function:

ResolveHandleOnMainThread() at ParticleFxHook.cpp:287-339:
Line 303-309: Calls g_find() (the engine's FindParticleSystem vtable method) with t_jitRedirect = true to upgrade any blocking load to a JIT-manifest load.
Line 309: FindParticleSystem call is the actual precache/load—it blocks if the system isn't already loaded.
Line 315-326: Validates the returned handle against HandleCreateReady() checks: ensures data pointer is non-null, live counter > 0, and fallback fields aren't -1 (the -1 indicates unloaded children).
Lines 329-334: Caches the result. Failed resolves are cached with a 5-second cooldown (line 334), not retried immediately.
The Engine Call:

g_find is vtable +0x78 of CParticleSystemMgr (ParticleFxHook.cpp:34): FindParticleSystem(manager, out_slot, name, 1).
This function owns the "not precached → blocking load" path (line 32-33 comment), which means:
If the system has been precached (loaded) into the resource system's registry, it returns immediately (cache hit).
If not, it calls the resource system's blocking load, which disk-reads the .vpcf_c file and all child systems recursively (see CRASH LESSON 6 and section 3 below).
JIT Manifest Redirect (Critical Optimization):

ParticleFxHook.cpp:198-206 (Hook_PlainBlockingLoad): While the resolver's FindParticleSystem call is active (detected via thread-local t_jitRedirect), any plain blocking load is redirected to BlockingLoadResourceByNameIntoJustInTimeManifest (resource system vt+0x148).
Without this redirect: The definition would load but child systems stay unresolved (fallback fields stay -1), causing engine crashes at particles.dll+0x3E3xx.
With this redirect: The manifest-load path ensures children are resolved and handle fields are fixed up before returning.
